# 🚁 Ловец-FPV — радар агентств недвижимости

Бот для FPV-пилота: раз в день присылает в Telegram несколько агентств
недвижимости Москвы с **готовым персональным КП** (Claude пишет письмо под
каждое агентство, со ссылкой на портфолио). Одна кнопка — и КП улетает
агентству на почту. Для Telegram/WhatsApp — кнопки с быстрым переходом.

Крутится целиком на GitHub Actions — компьютер не нужен. Основа —
[lovec-sjomka](https://github.com/safonovich/lovec-sjomka), но логика обратная:
там ловим заказы клиентов, тут сами выходим на клиентов.

## Как это работает

1. `data/agencies.json` — база агентств (стартовые 15 уже внутри, дополняй).
2. Workflow **Collect** (запуск вручную) обходит сайты агентств и достаёт
   email/телефон/Telegram.
3. Каждый день в ~10:00 МСК бот берёт 3 новых агентства из базы, тянет их
   сайт, генерирует персональное КП через Claude и присылает карточки в Telegram.
4. Кнопки под карточкой:
   - **📧 Отправить на email** — письмо уходит с твоей почты (текст ровно тот,
     что в карточке). Срабатывает при ближайшем прогоне, до ~15–25 мин.
   - **👎 Пропустить** — агентство помечается и больше не предлагается.
   - **✈️ Telegram / 💬 WhatsApp** — открыть чат агентства (в WhatsApp текст
     КП подставится сам, в Telegram — скопируй из карточки).

## Запуск с нуля (≈30 минут)

1. **Форкни/склонируй репозиторий** себе на GitHub. Делай **публичным** —
   у публичных Actions бесплатны без лимита.
2. **Создай бота**: @BotFather → `/newbot` → получишь токен.
   ⚠️ Напиши боту любое сообщение (Start), иначе он не сможет тебе писать.
3. **Узнай свой ID**: @userinfobot.
4. **Пароль приложения для почты**:
   - Яндекс: id.yandex.ru → Безопасность → Пароли приложений → «Почта»
   - Gmail: myaccount.google.com → Безопасность → двухэтапная проверка → Пароли приложений
5. **Секреты** (Settings → Secrets and variables → Actions):

| Секрет | Что это |
|---|---|
| `TG_BOT_TOKEN` | токен бота из BotFather |
| `TG_CHAT_ID` | твой Telegram ID |
| `SMTP_USER` | адрес почты, с которой шлём КП |
| `SMTP_PASS` | пароль приложения (не обычный пароль!) |
| `LLM_API_KEY` | ключ нейросети, опционально: Grok — console.x.ai, Claude — console.anthropic.com |

6. **Заполни `fpv/config.toml`**: ссылка на портфолио, имя, телефон.
   Для Gmail поменяй `smtp_host` на `smtp.gmail.com`.
7. **Собери контакты**: Actions → **Collect** → Run workflow. Проверь лог —
   он покажет, у кого нашлись email, а чьи сайты не ответили.
8. **Первый запуск**: Actions → **FPV** → Run workflow. Дальше — само по расписанию.

## База агентств

- **Пополняется сама**: workflow **Discover** каждый понедельник тянет
  агентства недвижимости Москвы из OpenStreetMap (бесплатно, без ключей) и
  добавляет до 30 новых за раз — только с сайтом или почтой. Можно запустить
  и вручную: Actions → Discover → Run workflow.
- Руками тоже можно: открыть `data/agencies.json` на GitHub → карандаш →
  дописать → Commit. Достаточно `id`, `name`, `site` — остальное добьёт Collect.
- Формат записи — в `fpv/models.py`.
- Кончились новые — бот сам напомнит в Telegram (и тогда просто запусти Discover).
- Статусы: `new` → `offered` (карточка выдана) → `sent`/`skipped`.
  Ответили на письмо — поменяй руками на `replied`, чтобы видеть конверсию.

## Лендинг-портфолио (GitHub Pages)

В `docs/index.html` лежит одностраничный сайт: КП, примеры работ, цены,
контакты. Видео он подтягивает напрямую из публичной папки Яндекс.Диска —
добавил ролик в папку, он сам появился на сайте.

Включить: Settings → Pages → Source: Deploy from a branch → main, папка
`/docs` → Save. Через пару минут сайт живёт на
`https://<твой-логин>.github.io/<имя-репо>/` — эту ссылку и ставь
в `portfolio_url` в `fpv/config.toml`.

Перед запуском в `docs/index.html`:
- замени `{{ИМЯ}}`, `{{ТЕЛЕФОН}}`, `{{TG_USERNAME}}`, `{{EMAIL}}` на свои
- подписи к роликам — в объекте `TITLES` в конце файла
- ⚠️ .mov-файлы играют не во всех браузерах: надёжнее перезалить на Диск
  те же ролики в MP4 (H.264). Если видео не грузится, страница сама покажет
  ссылку на папку Диска.

## Переговоры: что происходит после отправки КП

- **Агентство ответило** → бот видит письмо (проверяет ящик по IMAP каждые
  ~15 мин), Claude пишет черновик ответа и присылает карточку в Telegram:
  их письмо + черновик + кнопки **📤 Отправить ответ** / **✏️ Сам отвечу**.
  Ничего не уходит без нажатия кнопки. Ответ ложится в тот же email-тред.
- **Рамки для Claude**: цены только из прайса, без скидок и обещаний дат,
  цель ответа — получить адрес объекта или назначить созвон. Вопросы по
  договору, юридические и негатив — бот не отвечает, зовёт тебя.
- **Не ответили 4 дня** → бот сам шлёт одно вежливое напоминание
  (текст и срок — в `[followup]` в config.toml). Второго не будет.
- Дальше диалог живёт: каждый новый ответ агентства → новая карточка.

## Нейросеть: какая и почём

Провайдер настраивается в `[llm]` в config.toml. По умолчанию — **Grok
(xAI)**, модель grok-4.1-fast: $0.20/$0.50 за млн токенов, на наших объёмах
~1–2 $ в месяц. Подойдёт и любой другой OpenAI-совместимый API (DeepSeek,
OpenRouter, локальный LM Studio — поменяй `api_base` и `model`) или Claude
(`provider = "anthropic"`). Ключ — в секрет `LLM_API_KEY`.

**Без ключа тоже работает**: КП собирается из шаблона (без персонализации),
а ответы агентств пересылаются в Telegram без черновиков.

## Этика рассылки

- 3 письма в день — это точечный outreach, не спам-пушка. Не задирай
  `daily_leads`: массовость убивает доставляемость и репутацию почты.
- Письма уходят с твоего личного ящика — отвечай на ответы быстро.
- Если агентство просит больше не писать — ставь `skipped`.

## Настройка

`fpv/config.toml`: сколько лидов в день, час выдачи, рабочие часы, шаблон КП,
SMTP. Правится прямо на GitHub.
